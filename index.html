Git and Github Overview

Git is a free and open-source distributed version control system for tracking changes in source code and is used for software development.

GitHub is a Git repository hosting service. It provides a Web-based graphical interface, access control and collaboration features.

How to install Git and Github

To use Git on the command line, you'll need to download, install, and configure Git on your computer. Git can be downloaded through Git.After download, run the setup to install Git

You can install GitHub CLI to use GitHub from the command line GithubCLI. You can also use the Github Desktop to work with Github locally and without using command lines

You need to set your identity (username and email). Then create a repository on Github. Clone that repository into your local device. Finally, push the changes made locally online to the Github repository after working. Detail commands are below.

Set identity

Git config commands (username and email)

[$git config --global user.name "username"] then click enter to set username

[$git config --global user.email "email"] then click enter to set user email

Pushing changes online

Git add, commit and push command

[$git add .], enters then type [$git commit -m 'message']. After the command runs, the status of changes made will be shown. Finally, type [$git push] and enters to lock in the changes made into the online repository

Resolving conflict

Git pull command (git fetch and git merge)

Conflict happens when competing changes are made to the same line/ file. To solve, you must decide what to do with the changes (keep both, delete yours/other's ..). The git pull command is a combination of git fetch and git merge. Type [$git pull] and enters. First the remote repository will be downloaded to your local one (git fetch), then the conflict will be dectected and you will have choices on which changes to keep, after your selection your repo will be the same as the remote according to your choice (git merge)

Branching

Git branch command

branches are indentical copies of the source code. When you want to add a new change, work on a branch instead of the source will ensure safety for the original files if something goes wrong, it also make things cleaner and easier collaboration. type [$git branch "branch name"] to create a branch, [$git checkout "branch name"] to navigate between branches or come back to the source files.
